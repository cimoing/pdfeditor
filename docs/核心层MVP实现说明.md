# 核心层 MVP 实现说明

## 1. 实现范围

当前已创建 Rust workspace，并实现 `pdfeditor-core` 核心库。

核心库不依赖 UI 框架，不依赖浏览器运行时，不引入第三方依赖，适合作为后续桌面端、移动端或命令行工具的共享核心层。

## 2. 已实现模块

### 2.1 PDF 引擎抽象

文件：

- `crates/pdfeditor-core/src/engine.rs`

已定义：

- `PdfEngine`
- `EngineDocument`
- `MockPdfEngine`
- `LopdfEngine`

说明：

- `PdfEngine` 用于抽象真实 PDF 引擎。
- `MockPdfEngine` 用于 MVP 阶段跑通核心流程和测试。
- `LopdfEngine` 是当前接入的真实 PDF 后端，用于打开、解析、修改和保存基础 PDF 文本内容。
- 后续如需更强渲染和复杂 PDF 兼容性，可继续接入 PDFium 或 MuPDF。

## 5. 后续接入真实 PDF 引擎

下一步建议：

1. 基于 `LopdfEngine` 继续增强文本内容流兼容性。
2. 支持更多文本绘制操作、字体编码和压缩内容流场景。
3. 将 `MockPdfEngine` 继续保留用于单元测试。
4. 如需要高质量页面渲染和复杂 PDF 兼容性，可继续实现 `PdfEngine` for `PdfiumEngine`。
5. 再逐步接入图片替换、增量保存或更复杂的重写保存。

## 6. 当前限制

- `MockPdfEngine` 仅用于核心层流程验证，不是真实 PDF 编辑引擎。
- `LopdfEngine` 已接入真实 PDF 后端，当前支持简单 `Tj`、`TJ` 文本绘制操作的提取、替换和保存。
- 核心层已经支持修改已有文本对象，并能通过 mock 引擎保存可观察的修改结果。
- 文本修改策略限制为原文本对象原矩形范围内的内容替换；超出矩形范围的文本会被拒绝。
- 多样式文本通过 `TextRun` 表达，真实 PDF 后端需要将 runs 映射为对应的字体、颜色和文本绘制指令。
- 当前 `LopdfEngine` 不做页面位图真实渲染，只提供空白占位位图；后续如需真实渲染建议接入 PDFium 或 MuPDF。
- 当前 `LopdfEngine` 尚不处理复杂字体编码、文字重排、扫描件 OCR、真实多样式写回和图片对象落盘。
- 图片编辑命令已经建模，但真实图片对象落盘仍需要 PDF 引擎实现。
- 暂未实现缩略图缓存和后台任务队列。
